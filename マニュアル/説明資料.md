## 1. このアプリは何をするものか

このアプリは「チーム進捗ツール」です。
6人チームのタスク管理・進捗確認・ワークフロー管理をブラウザ上で行えます。

### 主な機能

| ページ | できること |
|--------|-----------|
| 名前選択 | 自分の名前を選んでダッシュボードに入る |
| ダッシュボード | チーム全体の進捗・KPI・メンバー別の状況を一覧で確認 |
| 自分のタスク | 担当タスクの確認・完了チェック・進捗率の更新 |
| ワークフロー | 業務の流れ（フロー）の確認・ボトルネックの管理 |
| タスク作成 | 新しいタスクをフォームで追加 |

## 2. 使っている言語・技術の説明

### 2-1. HTML（HyperText Markup Language）

一言で言うと：Webページの「骨組み」を作る言語。

料理で例えると「食器の形」です。
「ここにタイトルを置く」「ここにボタンを置く」という構造を定義します。

例：<button>ボタン</button> と書くと、画面にボタンが表示されます。

このアプリでは index.html という1ファイルだけがHTMLです。
他のファイルはすべて JavaScript で書かれています。

---

### 2-2. CSS（Cascading Style Sheets）

一言で言うと：Webページの「見た目・デザイン」を決める言語。

料理で例えると「盛り付け・色付け」です。
「ボタンの色を緑にする」「文字の大きさを14pxにする」といった装飾を担当します。

このアプリでは index.html の中に CSS が書かれており、
--accent: #06C755 という書き方でLINEグリーンの色を定義しています。
この「変数」を1箇所変えるだけで、アプリ全体の色が変わる仕組みになっています。

---

### 2-3. JavaScript（JS）

一言で言うと：Webページに「動き・機能」をつける言語。

料理で例えると「調理の手順」です。
「ボタンを押したらタスクが保存される」「チェックすると完了になる」といった
ユーザーの操作に応じた動作を担当します。

世界で最も使われているプログラミング言語の一つで、
ブラウザ（ChromeやSafariなど）なら追加インストール不要で動きます。

---

### 2-4. React（リアクト）

一言で言うと：JavaScriptをもっと使いやすくするための「道具箱」。

Facebookが作ったオープンソースのライブラリ（便利な道具の集まり）です。
「コンポーネント」という考え方で、画面をパーツ単位で組み立てられます。

■ コンポーネントとは？
レゴブロックのような「再利用できる部品」のこと。
例えば「メンバーのアイコン」を1回作れば、ダッシュボードにも、タスク画面にも
同じパーツを使い回せます。

このアプリがReactを使っている理由：
・タスクを更新したとき、画面の一部だけ自動で更新できる（ページ全体をリロードしなくていい）
・パーツを分けて作れるので、コードが管理しやすい
・世界中で使われており、情報が豊富

---

### 2-5. JSX（JavaScript XML）

一言で言うと：JavaScriptの中にHTMLのような書き方ができる特別な記法。

通常のJavaScriptはHTMLを直接書けませんが、JSXを使うと、
JSの中にHTMLに似た構文を混ぜて書けます。
ファイルの拡張子が .jsx なのはそのためです。

---

### 2-6. Babel（バベル）

一言で言うと：JSXをブラウザが理解できる形に「翻訳」するツール。

ブラウザはJSXを直接読めません。
BabelがJSXを普通のJavaScriptに変換してくれるので、
ChromeやSafariでも動くようになります。

---

### 2-7. localStorage（ローカルストレージ）

一言で言うと：ブラウザの中にデータを保存する「メモ帳」。

タスクを追加したり、進捗を更新したりすると、
そのデータがあなたのパソコンのブラウザ内に保存されます。

重要な制限：
・自分のパソコンのブラウザにしか保存されない
・他の人のパソコンには共有されない
・これが「今のままではチームで使えない」最大の理由

## 4. なぜファイルを分けているのか

「1つのファイルに全部書けばいいのでは？」と思うかもしれません。
しかし、それには大きな問題があります。

【分けない場合の問題】
・ファイルが10,000行以上になり、読めなくなる
・「ダッシュボードだけ直したい」のに、全体を探し回る必要がある
・複数人で作業するとき、同じファイルを同時に編集してぶつかる
・「このボタン、どこに書いてあったっけ？」が頻発する

【分けることの利点】
・「タスクを直したい」→ page-my-tasks.jsx だけ見ればいい
・「ロゴを変えたい」→ layout.jsx だけ触ればいい
・複数人が別々のファイルを同時に編集できる
・「このファイルは何をするファイル」がすぐわかる

【分け方の基準】
このアプリでは「役割ごと」に分けています：

分類        ファイル               例え
データ      data.js               倉庫
部品        ui-components.jsx     工具箱
枠組み      layout.jsx            建物の骨組み
各ページ    page-*.jsx            各部屋
全体管理    app.jsx               管理事務所

## 5. 現在の限界：なぜチームで使えないのか

現在のアプリには致命的な問題があります。

【問題：データが自分のブラウザにしか保存されない】

現在の仕組み：

  【じゅんのパソコン】          【あいるのパソコン】
    ブラウザ内のメモ帳              ブラウザ内のメモ帳
    （タスクAを保存）               （タスクBを保存）
          ↑                               ↑
     お互いに見えない！            お互いに見えない！

結果として起こる問題：
・じゅんがタスクを更新しても、あいるの画面には反映されない
・あいるのパソコンで開いても、じゅんのタスクは見えない
・ブラウザの履歴を削除するとデータが消える

【解決策：インターネット上の共有データベースにデータを置く】

理想の仕組み：

  【じゅんのPC】  【あいるのPC】  【けいのPC】
       |               |               |
       └───────────────┴───────────────┘
                       |
                       ↓
        【インターネット上の共有データベース（Supabase）】
             タスクA・タスクB・KPI・フロー...
                       ↑
           全員がここを見る！変更も全員に届く！

## 6. チームで使えるようにする手順（Supabase編）

### Supabaseとは？

Supabase（スーパーベース）とは、インターネット上にデータを保存できる
「無料のクラウドデータベース」サービスです。
難しいサーバーの設定なしに、データを共有できる仕組みを提供してくれます。

データベースとは？
Excelのような表（テーブル）にデータを整理して保存できる場所。
「タスク一覧」「メンバー情報」などを行・列で管理できます。

---

### Step 1：Supabaseアカウントを作る（約5分）

なぜやるのか：
チームのデータを保存する「倉庫」を借りるための登録です。無料で始められます。

操作手順：
1. ブラウザで https://supabase.com を開く
2. 右上の「Start your project」ボタンをクリック
3. GitHubアカウントまたはメールアドレスでサインアップ
4. メール認証を完了させる

---

### Step 2：プロジェクトを作成する（約3分）

なぜやるのか：
「チーム進捗ツール専用の倉庫」を作るためです。
Supabaseでは1アプリ = 1プロジェクトという単位で管理します。

操作手順：
1. ログイン後、「New project」ボタンをクリック
2. 以下を入力する：
   - Name（プロジェクト名）：teamflow など任意の名前
   - Database Password（パスワード）：強力なパスワードを設定し必ずメモしておく
   - Region（地域）：Northeast Asia (Tokyo) を選択
3. 「Create new project」をクリック
4. 2〜3分待つ（プロジェクトが作られる）

---

### Step 3：データを保存する「テーブル」を作る（約10分）

なぜやるのか：
「タスク情報を保存する表」「KPI情報を保存する表」など、
データの種類ごとに整理された棚（テーブル）を用意するためです。
Excelで言うと「新しいシートを追加して、列名を決める」作業です。

操作手順：
1. Supabaseのダッシュボード左メニューで「SQL Editor」をクリック
2. 「New query」をクリック
3. 以下のコードをコピー＆ペーストして「Run」ボタンを押す

---- ここからコピー ----

create table tasks (
  id text primary key,
  title text not null,
  assignee text,
  business text,
  project text,
  workflow_stage text,
  due_date text,
  progress integer default 0,
  status text default '未着手',
  memo text,
  priority text default '中',
  created_at text,
  completed_at text
);

create table kpis (
  id text primary key,
  title text not null,
  target_value integer,
  current_value integer default 0,
  unit text,
  related_project text
);

create table flows (
  id text primary key,
  title text not null,
  business text
);

create table steps (
  id text primary key,
  flow_id text references flows(id),
  title text,
  status text default '未着手',
  step_order integer default 0,
  assignees text[],
  description text
);

---- ここまでコピー ----

SQLとは？
データベースに「この表を作って」「このデータを取り出して」などと
命令するための専用言語です。英語に近い書き方で読めます。
create table tasks = 「tasksという名前の表を作る」という意味。

---

### Step 4：接続キーを確認する（約2分）

なぜやるのか：
アプリがSupabaseの「どのプロジェクトに」アクセスするかを示す「鍵」が必要だからです。
鍵がないと誰でもデータにアクセスできてしまいます。

操作手順：
1. Supabaseダッシュボードの左メニュー「Settings」→「API」をクリック
2. 以下の2つをメモしておく（後で使います）：
   - Project URL：https://xxxxxxxx.supabase.co という形式のURL
   - anon public key：長い英数字の文字列（eyJ... で始まる）

---

### Step 5：コードにSupabaseを接続する（約15分）

なぜやるのか：
アプリがlocalStorage（ブラウザのメモ帳）ではなく、
Supabaseのデータベースを読み書きするよう切り替えるためです。

今まで「自分の家のバケツ（localStorage）」に水を貯めていたのを、
「共有の水道（Supabase）」につなぎ替えるイメージです。

操作手順：

まず index.html の head タグの中に1行追加します：

  <script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2"></script>

次に data.js の先頭に以下を追加します
（xxxxxxxx の部分はStep4でメモしたあなたの情報に書き換えます）：

  const SUPABASE_URL = 'https://xxxxxxxx.supabase.co';
  const SUPABASE_KEY = 'eyJxxxxxx...';
  const db = supabase.createClient(SUPABASE_URL, SUPABASE_KEY);

  window.loadTasks = async function() {
    const { data } = await db.from('tasks').select('*');
    return data || [];
  };

  window.saveTasks = async function(tasks) {
    await db.from('tasks').upsert(tasks);
  };

async/await とは？
インターネット越しのデータ送受信には「時間」がかかります。
async/await は「データが届くまで待つ」という命令です。
郵便を送って返事が来るまで待つイメージです。

### Step 6：Vercelでインターネットに公開する（約10分）

なぜやるのか：
現在のアプリはあなたのパソコンの中にあります。
チームメンバーがアクセスするには、インターネット上に公開する必要があります。

Vercelとは？
作ったWebアプリを「URL付き」でインターネットに無料公開できるサービス。
https://teamflow.vercel.app のようなURLが発行されます。
お店を「開店」させるようなイメージです。

GitHubとは？
コードを保存・管理できるサービス。「コードのDropbox」のようなもの。
Vercelと連携して、GitHubに保存したコードを自動で公開できます。

【GitHubの準備】
1. https://github.com でアカウント作成
2. 「New repository（新しい保存場所）」を作成
   - Repository name（名前）：teamflow
   - 「Create repository」をクリック
3. webapp フォルダの中身を全部アップロード
   - 「uploading an existing file」リンクをクリック
   - ファイルをドラッグ＆ドロップ
   - 「Commit changes」をクリック

【Vercelでの公開】
1. https://vercel.com でアカウント作成（GitHubアカウントでログインが楽）
2. 「Add New Project」をクリック
3. GitHubの teamflow リポジトリを選択
4. 「Deploy」ボタンを押す
5. 2〜3分でURLが発行される（例：https://teamflow-xxx.vercel.app）

---

### Step 7：チームメンバーに展開する（約1分）

なぜやるのか：
URLをチームメンバーに共有するだけで、全員が同じデータを見ながら使えるようになります。

操作手順：
1. Vercelから発行されたURLをコピー
2. LINEやSlackでチームメンバーに送る
3. メンバーはURLをブラウザで開くだけで使える
4. 全員のデータが共有データベース（Supabase）に保存・同期される

LINEで送るメッセージ例：
「チーム進捗ツール作ったよ！
 https://teamflow-xxx.vercel.app
 ブラウザで開いて名前選択してね！
 スマホでも使えるよ！」

## 7. 用語集

HTML
  Webページの骨組みを作る言語。「ここにボタン」「ここにタイトル」という構造を定義する。

CSS
  Webページの見た目を決める言語。色・大きさ・配置などを指定する。

JavaScript (JS)
  Webページに動きをつける言語。ボタンを押したときの動作などを担当する。

JSX
  JavaScriptの中にHTMLっぽい書き方ができる特殊な記法。Reactで使う。

React
  Facebookが作ったJSの便利ツール。画面をパーツ単位で管理できる。

コンポーネント
  Reactにおける「再利用できる画面パーツ」。レゴブロックのようなもの。

Babel
  JSXを普通のJSに翻訳するツール。ブラウザが読めるように変換してくれる。

localStorage
  ブラウザに内蔵されたメモ帳。自分のパソコンにだけデータが保存される。

データベース
  データを表（テーブル）形式で整理して保存できる場所。Excelに似ている。

テーブル
  データベースの中の「表」。Excelのシートのようなもの。

Supabase
  無料で使えるクラウドデータベースサービス。チームでデータを共有できる。

SQL
  データベースに命令するための言語。「この表を作って」「このデータを取り出して」などと指示できる。

Vercel
  WebアプリをURLつきでインターネットに公開できる無料サービス。

GitHub
  コードを保存・管理できるサービス。コードのDropboxのようなもの。

API
  アプリ同士が通信するための「窓口」。SupabaseのAPIキーはその認証用の鍵。

クラウド
  インターネット上にある共有のサーバー（コンピューター）のこと。

async/await
  インターネット通信の「待つ」を表すプログラムの書き方。

CDN
  インターネット上にある共有の「図書館」。ReactやBabelをそこから借りてきている。

リアルタイム
  データが更新されたとき、すぐ全員の画面に反映される仕組み。

デプロイ
  作ったアプリをインターネットに公開すること。

ライブラリ
  誰かが作った便利な道具の集まり。Reactもライブラリの一種。

オープンソース
  コードが無料で公開されており、誰でも使える状態のこと。

リポジトリ
  GitHubでコードを保存する「フォルダ」のようなもの。
